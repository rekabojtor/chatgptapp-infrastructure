# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Terraform infrastructure for a ChatGPT-like app hosted on AWS (`eu-central-1`), served at `app.rekabojtor.com`. The entire infrastructure is defined in a single `main.tf` file with provider configuration in `terraform.tf`.

AWS profile used: `reka` (configured via `~/.aws/credentials` or `~/.aws/config`).

## Common Commands

```bash
terraform init        # Initialize providers (run after cloning or provider changes)
terraform plan        # Preview changes
terraform apply       # Apply changes
terraform destroy     # Tear down all resources
terraform output      # Show outputs (app URL, SSH command, name servers)
terraform output -raw api_token   # Reveal the sensitive API token
```

## Architecture

All resources live in a single flat `main.tf`. The stack:

- **Networking**: VPC (`10.0.0.0/16`) with two public subnets in `eu-central-1a/b`, an internet gateway, and a public route table.
- **Security groups**: ALB SG allows HTTPS (443) from `0.0.0.0/0`; EC2 SG allows port 8000 only from the ALB SG, and SSH (22) only from the deployer's current public IP (fetched dynamically via `data.http.my_ip`).
- **Compute**: Single `t2.micro` EC2 instance (Amazon Linux, `eu-central-1a`) running a backend on port 8000. SSH key loaded from `~/.ssh/mcp-ec2.pub`. The `API_TOKEN` is injected into `/etc/environment` via `user_data`.
- **Load balancer**: Application Load Balancer across both subnets → target group on port 8000 with `/health` health check → HTTPS listener with TLS 1.3 policy.
- **TLS & DNS**: ACM certificate for `app.rekabojtor.com` validated via Route 53 DNS records. Route 53 hosted zone with an A-alias record pointing to the ALB. The Route 53 name servers must be set at the domain registrar (visible via `terraform output route53_name_servers`).
- **API token**: Generated by `random_password` (32 chars, no specials), written to EC2 via `user_data`, and exposed as a sensitive Terraform output.

## Key Notes

- **State is local** (`terraform.tfstate`). Both `*.tfstate` files are gitignored by default but are currently committed — treat them as sensitive (they contain the plaintext API token).
- The ALB SG has a commented-out IP-restricted ingress rule (`cidr_blocks = ["${chomp(data.http.my_ip.response_body)}/32"]`). The active rule allows all IPs on 443.
- SSH access is dynamically restricted to the operator's current public IP via `data.http.my_ip`; running `terraform apply` from a different network will update the SSH security group rule.
- Replacing the EC2 instance regenerates `user_data` but does **not** rotate `random_password.api_token` unless that resource is also tainted/replaced.
